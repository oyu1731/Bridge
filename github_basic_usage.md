# GitHubの基本的な使い方

GitHubは、Gitというバージョン管理システムを利用して、コードの変更履歴を管理したり、複数人での共同開発を効率的に行うためのプラットフォームです。

## よく使うGitコマンド

ここでは、GitHubを使った開発で頻繁に利用するGitコマンドをまとめます。

### 1. リポジトリのクローン (git clone)

リモートリポジトリ（GitHub上）のコードをローカル環境にコピーします。

```bash
git clone [リポジトリのURL]
```

例: `git clone https://github.com/ユーザー名/リポジトリ名.git`

### 2. ブランチの作成と切り替え (git checkout)

ブランチは、メインのコードの流れから分岐して、新しい機能開発やバグ修正を行うためのものです。

- **新しいブランチの作成と切り替え**:
  ```bash
  git checkout -b [ブランチ名]
  ```
  例: `git checkout -b feature/new-login`

- **既存のブランチへの切り替え**:
  ```bash
  git checkout [ブランチ名]
  ```
  例: `git checkout main`

### 3. 変更のステージング (git add)

変更したファイルをコミットの対象として準備します。

- **すべての変更をステージング**:
  ```bash
  git add .
  ```

- **特定のファイルをステージング**:
  ```bash
  git add [ファイル名]
  ```
  例: `git add src/main.js`

### 4. コミット (git commit)

ステージングした変更を履歴として記録します。

```bash
git commit -m "[コミットメッセージ]"
```

例: `git commit -m "feat: ログイン機能を追加"`

### 5. プッシュ (git push)

ローカルリポジトリのコミットをリモートリポジトリにアップロードします。

```bash
git push origin [ブランチ名]
```

例: `git push origin feature/new-login`

### 6. プル (git pull)

リモートリポジトリの最新の変更をローカルリポジトリにダウンロードします。

```bash
git pull origin [ブランチ名]
```

例: `git pull origin main`

## GitHubの基本的な概念とワークフロー

### 1. GitHubとは

- **バージョン管理**: コードの変更履歴を記録し、いつでも過去の状態に戻せるようにします。
- **共同開発**: 複数の開発者が同じプロジェクトで作業する際に、コードの統合や競合の解決を支援します。
- **オープンソース**: 世界中の開発者がコードを公開し、貢献し合う文化を支えています。

### 2. リポジトリの作成

プロジェクトのコードを管理する場所を「リポジトリ」と呼びます。

1. GitHubのウェブサイトで「New repository」をクリック。
2. リポジトリ名を入力し、公開/非公開を選択。
3. 「Create repository」をクリック。

### 3. プルリクエスト (Pull Request) の作成

自分のブランチでの変更を、メインのブランチ（例: `main`）に統合してもらうために提案します。

1. 自分のブランチをGitHubにプッシュ。
2. GitHubのウェブサイトで「Compare & pull request」をクリック。
3. 変更内容を確認し、レビュー担当者を指定してプルリクエストを作成。

### 4. マージ (Merge)

プルリクエストが承認された後、変更をメインのブランチに統合します。

1. プルリクエストページで「Merge pull request」をクリック。
2. 必要に応じてコミットメッセージを編集し、「Confirm merge」をクリック。
